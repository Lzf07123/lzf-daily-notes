# 经验总结（2026-06-24）

## 正确的事

1. **先 brainstorming 再动手** — 通过一轮轮问答明确了风格（童话绘本）、构图（左右分割）、色调（暖色），避免返工。
2. **设计规格文档先行** — 将确认方案写入 `docs/superpowers/specs/` 存档，交付物和设计依据可追溯。
3. **交付物分层** — 三个文件各司其职：提示词（生成背景）、排版指引（叠加文字）、快速上手（总览），不混在一起。
4. **逐条核查** — 完成后对着 README 逐条对照，发现了方向错误。**核查环节必须有。**

## 踩过的坑

1. **尺寸方向读错** — README 写「高60cm宽40cm」是竖版，我惯性理解为横版 60×40。所有文件中的「横版」「3:2」「3600×2400px」「ar 3:2」都需要修正。
   - **教训**：动手前先把关键参数（尺寸、方向、比例）高亮确认，不靠惯性判断。
