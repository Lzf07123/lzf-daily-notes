#### 💡 装完五个组件发现的一个规律

Keystone、Glance、Neutron、Placement、Nova——五个组件装下来，发现它们的配置文件都是同一种套路，就三个部分：

1. 🔑 **认证** — `[keystone_authtoken]` 这一段每个组件都有，告诉组件去哪找 Keystone 验身份。
2. 🗄️ **数据库** — `[database]` 这一段，告诉组件连哪个 MySQL 库。
3. ⚙️ **组件自己特有的配置** — 这部分每个组件不一样：Neutron 是 `[ml2]`/`[linux_bridge]`，Glance 是 `[glance_store]`，Nova 是 `[vnc]`/`[libvirt]`，Cinder 是 `[lvm]`。

所以以后学新组件，先把这三个部分的框架搭好再填具体值，不用死记每个配置项在哪。

> 🟢 **7/23补充：Neutron 和 Nova 已练熟。** 两个组件的部署流程和配置文件结构已经不需要对着文档敲了——Neutron 关键是网桥名别写错，Nova 关键是配置段多别漏。这两个可以进入计时练习阶段，腾出精力冲剩下的 Cinder、Swift、ISCSI。

#### ⚠️ 踩过的坑

- Keystone：`/etc/my.cnf.d/openstack.cnf` 路径容易写错，先确认目录存不存在。
- Glance：`filesystem_store_datadir` 要先建好再 chown 给 glance 用户。
- Neutron：网卡名必须跟 `ip a` 一致，别猜。
- Nova：配置段太多容易漏，对着清单逐段检查。

### 📋 对照2025-2026省赛任务书的差距（7月23日）

[↑ 目录](#toc)

拿到了一份2025-2026学年广东省赛的赛题总结，四大模块。对照了一下，有几个重要的发现：

**最关键的一条：考试用 iaas-\* 脚本部署，不是全手写。**

之前一直练"无文档从零写配置文件"，这个方向对（深入理解），但比赛实际提供 iaas-yoga 脚本包。训练策略要调整——**脚本快速部署（练速度）+ 手动改调优参数（练理解）+ 定时无文档部署（练独立）**，三样都要。

**当前计划完全没覆盖的考点：**

- Dashboard/Horizon 部署和定制（改标题、Django文件存储）
- ISCSI 存储配置（注意：考试是 ISCSI 不是 Ceph！）
- Heat 模板编排
- Cgroup 资源限制
- 各组件的调优参数（Glance缓存、Nova调度器/并发限制、Keystone memcached、Etcd配额、K8s控制器并发）
- kubeVirt（K8s里跑虚拟机）
- Dify 大模型应用平台
- GitLab Agent（当前只计划了 Runner）
- 边缘计算的 Device/DeviceModel 定义和 CRUD 开发
- Streamlit 聊天机器人（之前计划是 Gradio，考试用 Streamlit）

**Python 开发任务比预期重很多，散落在四个模块里：**

| 模块      | 开发任务                                                       |
| :-------- | :------------------------------------------------------------- |
| OpenStack | MariaDB操作类 + 用户管理CLI（FastAPI+argparse）                |
| K8s       | Docker API管理 + 集群资源监控API（FastAPI）                    |
| 边缘计算  | AI人脸识别服务（FastAPI） + 设备管理CRUD（FastAPI+K8s SDK）    |
| LLM       | 智能聊天机器人（Streamlit+Ollama，多轮对话+文件解析+模型切换） |

Python 不能再等到 9 月了——OpenStack 模块本身就有两个开发任务，得在练部署的同时穿插着写。

**ISCIS vs Ceph：** 考试考 ISCSI（计算节点做 target，controller 做客户端），不考 Ceph。7月冲刺里把 Ceph 换成 ISCSI。

**当前进度：7/23，Keystone、Glance、Neutron、Nova 四个组件已练熟。** 剩下 Cinder、Swift、ISCSI 三天搞定，时间够。Neutron 和 Nova 可以进入计时练习，腾出精力冲新考点。

## 六、AI 辅助学习方法论

[↑ 目录](#toc)

[↑ 目录](#toc)

> 🎯 **目标：** 让 AI（Claude/Copilot/Cursor）成为你的"私人导师"，而不是替你完成作业的工具。  
> 📌 基础较弱的初学者，AI 是降低学习曲线最有效的杠杆——不用等于放弃这个时代最大的加速器。

### 9.1 五大使用场景

[↑ 目录](#toc)

| 场景                | 做什么                                                              | 原则                           |
| :------------------ | :------------------------------------------------------------------ | :----------------------------- |
| 🔧 **报错排错**     | 贴报错给 AI → 理解根因 → **自己手打命令修复**（不复制粘贴）→ 写笔记 | AI 解释为什么，你来动手        |
| 📖 **英文文档阅读** | 先自己读一遍→贴给 AI 翻译→对照差异→关掉翻译自己复述                 | AI 是拐杖，不是替代品          |
| 🏗️ **架构学习**     | 让 AI 生成 Mermaid 流程图（组件交互、调用链）→ 加到笔记             | 可视化加速理解                 |
| 💻 **代码生成**     | 让 AI 写框架/模板/YAML → **自己调试+理解每一行**                    | AI 写 70%，你补 30% + 全盘理解 |
| 🗣️ **英语学习**     | 用 AI 改英文写作语法、练英文技术对话                                | 把 AI 当语法老师               |

### 9.2 AI 使用红线

[↑ 目录](#toc)

| 不能替代的能力       | 为什么                         |
| :------------------- | :----------------------------- |
| 自己打命令的肌肉记忆 | 线上排错不能靠复制粘贴         |
| 裸读英文报错         | 面试没有 AI 翻译               |
| 原创内容的思考过程   | 博客/文档/排错笔记必须出自你手 |
| 脱离 AI 独立干活     | 每周至少半天不用 AI——防止依赖  |

> 📌 **一句话原则：AI 解释问题、生成框架，但决策和动手必须是你自己。**

## 七、英语突破路线图（技术向）

[↑ 目录](#toc)

[↑ 目录](#toc)

### 10.1 核心原则

[↑ 目录](#toc)

- 以"能读英文技术文档"为先，不以 CET-4 为目标

- **赛后（2027.01起）全力突击：每天 45-60 分钟**——专升本应试英语 + 技术英语并行
- Claude 是双语老师，但不能替代你读英文的过程

### 10.2 三阶段计划（2027年1月起正式启动，与专升本备考并行）

[↑ 目录](#toc)

> 📌 **备赛期（2026.07–12）缓慢积累：** 每天 ≥20min Anki技术单词 + 读英文报错不翻译。赛后只剩2个月备考专升本，不能从零起步。

| 阶段              | 时间                              | 核心动作                                                     | 里程碑                               |

### 10.3 应试英语（按需启动）

[↑ 目录](#toc)

专升本英语 ≈ CET-3~4 之间。技术英语训练已覆盖阅读核心能力，考前 2 个月集中刷真题即可：考前 3 月补词汇 → 考前 2 月刷真题 ×2 遍 → 考前 1 月背作文模板。

**长期：** 每天刷 Hacker News + 通勤听 K8s Podcast + 每年 1 次英文技术交流 + CKA 直接考英文版。

> ⚠️ **英语学习的最大陷阱：** 总想"等英语好了再看英文文档"。**你永远不会觉得自己英语足够好。** 正确做法是：从第 1 天就用英文文档，用痛苦驱动学习。

## 八、持续性与身心健康

[↑ 目录](#toc)

[↑ 目录](#toc)

> 16 年马拉松最怕 burnout。**停下休息 ≠ 落后，burnout 躺平半年才是。**

| 维度        | 规则                                                                                                                          |
| :---------- | :---------------------------------------------------------------------------------------------------------------------------- |
| 🏃 **运动** | 备赛期隔天20分钟（跑步/跳绳/俯卧撑）；周六下午+周日必须出门一次                                                               |
| 💤 **休息** | **周六下午+周日全天硬休息，不可挪用。** 期末后休息1周；每天睡前1小时不看屏幕；每天固定学习时段（如14:00–18:00），不熬夜补进度 |
| 🧠 **心理** | 连续两天不想碰技术→第三天无条件休息；频繁焦虑/失眠→找学校心理咨询师                                                           |
| 📵 **离线** | 周六下午和周日不打开 PVE、不写代码、不看笔记。如果忍不住学了——下周一补休半天                                                  |
| 😴 **作息** | 凌晨补笔记的习惯可以保留，但每天保证7小时睡眠。睡眠不足的那天学习效率砍半，不如早点睡                                         |

> 之前计划里的"弹性日"容易被用来补进度，等于没有休息。现在改成**周六下午+周日全天固定休息**——一周学5.5天休1.5天，比七天连轴转续航强得多。休息不是偷懒，是让脑子有时间消化。装了一周 OpenStack，周末歇一歇，周一回来效率反而更高。
>
> 当出现"一个 bug 修不好=人生完了"的想法时——停手，找朋友聊天。这是认知扭曲，不是真相。

## 十、定期评估与修正机制

[↑ 目录](#toc)

[↑ 目录](#toc)

> 📌 **核心理念：** 规划不是写完就锁死的——它是一个**活的文档**，需要定期校准。15 年蓝图靠的不是一次做对，而是持续做对。

### 12.1 四层回顾节奏

[↑ 目录](#toc)

|      层级       | 频率                | 耗时    | 核心动作                                                                         |
| :-------------: | :------------------ | :------ | :------------------------------------------------------------------------------- |
| ⚪ **每周微检** | 每周日              | 15 分钟 | 对照周计划检查完成率；在不偏离月目标的前提下微调下周任务；一句话记录本周最大卡点 |
| 🔵 **月度轻检** | 每月最后周末        | 30 分钟 | 对照月计划检查完成率；记录本月最大收获和最大卡点；微调下月计划                   |
| 🟡 **季度中检** | 每 3 个月           | 2 小时  | 检查里程碑达成情况；评估技术栈是否需调整；审视英语/高数/GitHub 进度              |
| 🔴 **年度深检** | 每年 6 月（暑假前） | 半天    | 对照"一张图看完"的时间线；重评 A/B 轨选择；更新薪资预期；大改年度计划            |

> 💡 **备赛期建议：** 每周日晚花15分钟做微检——只问三个问题：① 本周实际完成了什么？（不是"学了多少"，是"产出了什么"）② 和月目标偏离了多少？③ 下周最该砍掉/推迟什么？

> 💡 **为什么年度深检定在 6 月？** 因为你在读书——6 月是学年结束、暑假开始，最适合做大规模方向调整。

### 12.2 事件触发 + 评估维度 + 修正框架

[↑ 目录](#toc)

**事件触发（48h 内专项回顾）：** 比赛结果出炉 / 专升本成绩 / 拿到或失去实习 offer / 跳槽换岗 / 重大技术变革。

**季度/年度评估四问：** ① 技能：过去 3 个月真正变强的能力是什么？（不是"学过"，是"能用"）② 方向：A/B 轨和城市策略还成立吗？③ 状态：在成长还是假装忙碌？还有热情吗？④ 财务：存款进度？自我提升的投入回报？

**三级偏差响应：** 🟢 轻微→微调下月 / 🟡 中度→调节奏 / 🔴 重大→重写相关章节。

> ⚠️ **铁律：** 修正计划不是失败。承认太乐观并调整，比硬撑到 burnout 强 100 倍。
