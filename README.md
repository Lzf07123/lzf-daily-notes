## 📝 笔记怎么写

模板是四段式：`Context` → `Key Takeaway` → `Practice` → `Next Step`。具体看 [note-example.md](note-example.md)。

写了大半个月，几个心得：

**① Next Step 别空着。** 这个字段最容易跳过去，但它其实最有用——逼自己在收尾时想清楚明天干什么，这样笔记之间能串起来，不会变成每天孤立的一篇。

**② 踩坑记下来比贴配置有用。** 部署笔记很容易写成配置文件的复制粘贴。但一个月后回看，最有价值的往往是一句话："当时这里出了什么问题，怎么解决的"。格式就一句话：`> ⚠️ 踩坑：问题 → 解决`。

**③ 每周花几分钟扫一眼。** 周日晚上翻翻这周的笔记，看看 Next Step 都写了吗，有没有漏踩坑，有没有能提炼出来的规律。比如装完五个组件后发现它们配置文件都是同一种套路——这种总结只有回看的时候才会冒出来。
